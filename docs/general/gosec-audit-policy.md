# NeoAnvil `//nolint:gosec` Audit Rules

_Generated by Épica 234 on 2026-04-19._

Este documento registra las clases de `//nolint:gosec` suppressions legítimas
en el repositorio. Sirve como revisión de seguridad estática: cada vez que
gosec marca un path, el desarrollador tiene dos opciones — corregir el
código, o añadir un `//nolint:gosec // <CATEGORÍA>: <control compensatorio>`
basado en este documento.

**Prohibido:** `//nolint:gosec` sin categoría + control compensatorio. El
lint pasa pero la deuda queda oculta.

---

## Categorías validadas

### G304 — Potential file inclusion via variable
_gosec regla:_ `os.ReadFile(pathVar)` cuando `pathVar` no es literal.

**Válido cuando se cumple UNO de:**

- **`G304-WORKSPACE-CANON`** — Path calculado con `filepath.Join(workspace, ...)`
  + `filepath.Clean`, donde `workspace` está pinned al boot y no deriva de
  input del usuario. El handler valida que el Join no escape del workspace
  root (via prefix check o canonicalizado explícito).
  - Ejemplos: `radar_handlers.go` → `os.ReadFile(absTarget)` donde `absTarget`
    viene de `filepath.Join(t.workspace, target)` + validación.
  - Ejemplo: `pkg/state/planner.go` → comentario existente "workspace is
    strictly enforced" (se mantiene).

- **`G304-DIR-WALK`** — Path proviene de `os.ReadDir` + `filepath.Join` sobre
  un directorio cerrado bajo control del proceso (e.g. `.neo/incidents/`).
  La fuente del nombre de archivo es el filesystem, no el usuario.
  - Ejemplos: `pkg/incidents/indexer.go` → lecturas de INC-*.md en
    `.neo/incidents/`.

- **`G304-CLI-CONSENT`** — CLI tool donde el path viene del flag del usuario
  que opera el binario localmente. La suposición es que el operador puede
  leer lo que quiera ejecutando como sí mismo.
  - Ejemplos: `cmd/neo/main.go` → `os.ReadFile(s.src)` para carga de
    snapshots, `os.Create(outPath)` para output.

**NO VÁLIDO:**
- Path derivado de request HTTP externo sin validación.
- Path derivado de payload de plugin sin canonicalizado.
- Casos donde `workspace` pudiera ser configurable post-boot (violaría el
  contrato de workspace-pinned).

### G204 — Subprocess launched with function call as argument
_gosec regla:_ `exec.Command(var, ...)` cuando el binario no es literal.

**Válido cuando se cumple UNO de:**

- **`G204-LITERAL-BIN`** — El primer argumento (binario) ES literal, y los
  args subsiguientes son variables validadas (e.g., `exec.Command("git",
  "-C", workspace, "log", "-1", ...)`). gosec falso-positivo por detectar
  args variables.
  - Ejemplos: `radar_handlers.go:2069` → `exec.Command("git", "-C", workspace, ...)`
  - Ejemplos: `macro_tools.go` → `exec.CommandContext(ctx, "go", "build", pkgArg)`

- **`G204-SHELL-WITH-VALIDATION`** — `exec.Command("sh", "-c", cmd)` donde
  `cmd` es construido a partir de valores validados contra una allow-list
  o inputs de config pinned al boot.
  - Ejemplos: `radar_handlers.go:2140` → lint shell invocation — el comando
    viene de `cfg.SRE.LinterCmd` (config validada).
  - Ejemplos: `macro_tools.go:953` → `exec.CommandContext(ctx, "sh", "-c", buildCmd)`.

**NO VÁLIDO:**
- Comandos construidos por string-concat con input del usuario.
- Comandos con binarios variables derivados de request externa.

### G107 — URL provided to HTTP request as taint input
_gosec regla:_ `http.Get(url)` cuando `url` no es literal.

**Válido cuando se cumple UNO de:**

- **`G107-WRAPPED-SAFE-CLIENT`** — La llamada va a través de
  `sre.SafeHTTPClient()` o `sre.SafeInternalHTTPClient()` que ya bloquean
  SSRF. gosec no lo detecta porque analiza `http.Get` directamente.
  - Ejemplo: `pkg/nexus/fleet.go:135` → `client.Get(url)` con client de
    `SafeHTTPClient()`.

- **`G107-TRUSTED-CONFIG-URL`** — URL proviene de `neo.yaml` (operator
  controlled) y apunta a un endpoint interno documentado (Ollama, etc.).
  - Ejemplo: `cmd/neo/main.go:1091` → CLI, URL es env var `NEO_OLLAMA_URL`.

### G402 — TLS InsecureSkipVerify
_gosec regla:_ `InsecureSkipVerify: true` en `tls.Config`.

**Válido solo en:**

- **`G402-STRESS-TEST`** — Binarios de stress testing (`cmd/stress`,
  `cmd/plc_chaos`, `cmd/chaos`) que prueban infraestructura interna donde
  la identidad del servidor NO es una propiedad bajo test.

**NO VÁLIDO** en cualquier componente productivo.

### G306 — Write file with too-open permissions
_gosec regla:_ `os.WriteFile(path, data, 0666+)`.

**Válido solo cuando:**

- **`G306-SHARED-SOCKET`** — Archivo expuesto por diseño (socket Unix
  compartido entre procesos legítimos). Debe combinarse con `os.Chmod(0600)`
  post-Listen (LEY 5 del codebase).

## Control estandarizado

Cuando aparezca un nuevo hit de gosec al editar código:

```go
// Bad — supresión opaca
data, _ := os.ReadFile(path) //nolint:gosec

// Good — categoría + control documentado
data, _ := os.ReadFile(path) //nolint:gosec // G304-WORKSPACE-CANON: absPath via filepath.Join(t.workspace,...) y Clean, no escape posible
```

## Auditoría periódica

Cada PILAR debería ejecutar:

```bash
grep -rn "//nolint:gosec\s*$" --include="*.go" . | wc -l
```

y exigir que el conteo de "bare" (sin categoría) no aumente. La meta a
medio plazo es `0 bare` — todas las supresiones anotadas.
